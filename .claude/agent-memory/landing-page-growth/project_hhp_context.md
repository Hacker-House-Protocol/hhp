HHP is a pre-launch Web3 builder-matching platform. The landing page goal is waitlist email capture only — no auth, no product access yet.

**Why:** Pre-MVP. Privy API keys and project email are still pending. Waitlist emails go to Supabase table `waitlist`.

**How to apply:** Never design for conversion beyond email capture. Never reference metrics or community size (none exist yet). Never imply the product is live.

Key facts:
- Domain: hackerhouse.app
- Stack: Next.js App Router + Tailwind CSS v4 + Supabase + Privy (pending keys)
- Route `/` is the landing. Route `/home` is post-login app.
- Always dark — no light mode, ever.
- Language: English for all public-facing copy.
- One real upcoming event: ETH Global Cannes 2026 — a team member will be there.
- 100% free in MVP — no pricing, no plans.
- No direct competitors — do not position against another product.
- Three user archetypes with specific hex colors:
  - Visionary #990070 (has the idea)
  - Strategist #8B78E6 (connects resources)
  - Builder #6EE76E (ships the tech)
- Cypher Kittens: animated GIF avatars chosen at onboarding — not yet in repo.
